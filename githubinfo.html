All About Github

  • Q. What is a version control system?

  • A version control system in Github tracks the history of changes in a project. As people make changes to the project, the older versions can be recovered at any time.

  • Q. What is Git?

  • Git is a free and open source version control system.

  • Q. What is Github?

  • Github is a platform and cloud-based service for software development and version control that uses Git, allowing people to store and manage code.

  • Q. What is the difference between Git and Github?

  • Git is the version control system that allows developers to track changes in their code, and Github is a hosting service for these Git repositories.

  • Q. Who started Github and how was it started?

  • Chris Wanstrath, P. J. Hyett, Tom Preston-Werner, and Scott Chacon formed Github in 2007 and released it in early 2008, their purpose was to provide developers with the option to host code securely and manage commits to code securely.

  • Q. What company owns it now?

  • Microsoft Corp owns Github now.

  • Q. How much does a Github account cost?

  • Github account is free, but they do offer plans depending on what benefits a developer may need.

  • Q. What is the Octocat?

  • The Octocat is Github's mascot, it is a cat with five octopus-like arms created by graphic designer Simon Oxley.

    • Git/GitHub terms:

  • Repository

  • Encompasses the entire collection of files and folders associated with a project, along with each file's revision history.

  • Commit

  • Snapshots in time that contain the file history.

  • Fork

  • A new repository that shares code and visibility settings with the original “upstream” repository.

  • Push

  • Updates the remote branch with local commits.

  • Pull requests

  • Allows developers to tell others about changes they've pushed to a branch in a repository on Github.

  • Workflows

  • A configurable automated process that will run one or more jobs.

  • Issues

  • Allows developers to track their work on Github.

  • Raw button

  • Allows developers to view or copy the raw content of a file without any styling.

  • Blame button

  • Allows developers to view the line-by-line revision history, with the code in a file separated by commit.
